================================================================================
                         WHAT IS AN INDEX?
================================================================================

Imagine ek library mein 1000 books hain aur tumhe "Python Programming" book 
chahiye. Bina index ke, tumhe har book ek ek kke check karna padega - ye hai
"FULL TABLE SCAN". 

Lekin agar library mein ek index book hai jisme books ki list alphabetically 
hai, to tum directly "P" wale section mein jaa sakte ho - ye hota INDEX USE!

Database indices bilkul ussi tarah kaam karte hain.

================================================================================
                    INDICES IN THIS FINTECH APP
================================================================================

--------------------------------------------------------------------------------
1. idx_users_email
--------------------------------------------------------------------------------
   Table: users
   Column: email
   
   Kya karta hai: Jab user email se login karta hai, ye index use hota hai.
   Example: user@example.com se login karne ke liye fast search
   
   WITHOUT INDEX: 1 second (full table scan)
   WITH INDEX: 0.01 second (direct lookup)

--------------------------------------------------------------------------------
2. idx_users_mobile
--------------------------------------------------------------------------------
   Table: users  
   Column: mobile
   
   Kya karta hai: Mobile number se user dhundne ke liye
   Use case: Phone number based authentication

--------------------------------------------------------------------------------
3. idx_bank_accounts_user
--------------------------------------------------------------------------------
   Table: bank_accounts
   Column: user_id
   
   Kya karta hai: User ke saare bank accounts dhundne ke liye
   Example: User ki linked bank accounts dikhana

--------------------------------------------------------------------------------
4. idx_market_assets_type
--------------------------------------------------------------------------------
   Table: market_assets
   Column: asset_type
   
   Kya karta hai: Assets ko type (STOCK, CRYPTO, MUTUAL_FUND, GOLD) ke hisab 
                  se filter karne ke liye
   
   Example: Sirf "CRYPTO" wale assets dikhane ke liye
   SELECT * FROM market_assets WHERE asset_type = 'CRYPTO';

--------------------------------------------------------------------------------
5. idx_user_investments
--------------------------------------------------------------------------------
   Table: user_investments
   Columns: (user_id, asset_id)
   
   Kya karta hai: User ki specific investment dhundne ke liye
   Example: User ki Reliance Industries mein investment dekho

--------------------------------------------------------------------------------
6. idx_income_user_date
--------------------------------------------------------------------------------
   Table: income
   Columns: (user_id, date)
   
   Kya karta hai: User ki income ko date range ke hisab se dhundne ke liye
   
   Example: "January 2024 ki saari income dikhao"
   SELECT * FROM income WHERE user_id = 1 AND date BETWEEN '2024-01-01' AND '2024-01-31';

--------------------------------------------------------------------------------
7. idx_expenses_user_date
--------------------------------------------------------------------------------
   Table: expenses
   Columns: (user_id, date)
   
   Kya karta hai: User ki expenses ko date range ke hisab se dhundne ke liye
   
   Example: "Last 7 days ki expenses dikhao"
   SELECT * FROM expenses WHERE user_id = 1 AND date >= date('now', '-7 days');

   YEH SABSE IMPORTANT INDEXES MEIN SE EK HAI!

--------------------------------------------------------------------------------
8. idx_expenses_category
--------------------------------------------------------------------------------
   Table: expenses
   Columns: (user_id, category, date)
   
   Kya karta hai: Category-wise expenses analytics ke liye
   
   Example: "Food & Dining pe kitna kharcha kiya?"
   SELECT * FROM expenses WHERE user_id = 1 AND category = 'Food & Dining';

--------------------------------------------------------------------------------
9. idx_budgets_user
--------------------------------------------------------------------------------
   Table: budgets
   Columns: (user_id, year, month)
   
   Kya karta hai: User ki specific month ki budget dhundne ke liye
   
   Example: "January 2024 ki budget dekho"
   SELECT * FROM budgets WHERE user_id = 1 AND year = 2024 AND month = 1;

--------------------------------------------------------------------------------
10. idx_goals_user
--------------------------------------------------------------------------------
    Table: financial_goals
    Columns: (user_id, status)
    
    Kya karta hai: User ke active goals dhundne ke liye
    
    Example: "Mere saare active goals dikhao"
    SELECT * FROM financial_goals WHERE user_id = 1 AND status = 'ACTIVE';

--------------------------------------------------------------------------------
11. idx_wallet_txn_user
--------------------------------------------------------------------------------
    Table: wallet_transactions
    Columns: (user_id, date)
    
    Kya karta hai: User ki wallet transactions date-wise dhundne ke liye
    
    Example: "Wallet transaction history dekho"
    SELECT * FROM wallet_transactions WHERE user_id = 1 ORDER BY date DESC;

--------------------------------------------------------------------------------
12. idx_audit_actor
--------------------------------------------------------------------------------
    Table: audit_logs
    Columns: (actor_type, actor_id)
    
    Kya karta hai: Admin ya User ki saari actions ki logs dhundne ke liye
    
    Example: "Admin #5 ne kya kya actions kiye?"
    SELECT * FROM audit_logs WHERE actor_type = 'ADMIN' AND actor_id = 5;

--------------------------------------------------------------------------------
13. idx_audit_date
--------------------------------------------------------------------------------
    Table: audit_logs
    Column: created_at
    
    Kya karta hai: Date range ke hisab se logs dhundne ke liye
    
    Example: "Aaj ki saari logs dekho"
    SELECT * FROM audit_logs WHERE date(created_at) = date('now');

--------------------------------------------------------------------------------
14. idx_notifications
--------------------------------------------------------------------------------
    Table: notifications
    Columns: (user_id, is_read)
    
    Kya karta hai: User ke unread notifications dhundne ke liye
    
    Example: "Mere kitne unread notifications hain?"
    SELECT * FROM notifications WHERE user_id = 1 AND is_read = 0;

================================================================================
                         PERFORMANCE COMPARISON
================================================================================

Scenario: 1 million records ke expenses table se January 2024 ki expenses 
          dhundhni hain

WITHOUT INDEX:
- Time: ~5-10 seconds
- Process: Har ek row check karega (FULL TABLE SCAN)
- CPU: High usage

WITH INDEX (idx_expenses_user_date):
- Time: ~0.01-0.05 seconds  
- Process: Direct index se record mil jayenge
- CPU: Low usage

SPEED IMPROVEMENT: 100x to 1000x faster!

================================================================================
                         SUMMARY
================================================================================

Index Type              | Purpose
------------------------|------------------------------------------------
User Lookups           | email, mobile se user dhundna
Date Range Queries      | income, expenses, wallet transactions
Category Filters        | expenses by category, assets by type
User-specific Data      | budgets, goals, investments per user
Analytics              | category-wise spending analysis
Notifications          | unread notifications
Audit/Logs             | action history tracking

Indices database ki "speed boost" hain - har query ko fast banate hain 
especially jab data grow karta hai!
